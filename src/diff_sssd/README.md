# SynthesizedECG

## Current target module
- `src/diff_sssd`: diffusion-based ECG pretrain + sample synthesis flow using SSSD-ECG.

## Folder map
- `src/diff_sssd/codes`: training framework (train loop, trainer classes, dataloader/model wrappers).
- `src/diff_sssd/sssd_standalone`: bundled standalone SSSD-ECG implementation used by both training and synthesis.
- `src/diff_sssd/resources/exp_yamls`: experiment YAMLs for training.
- `src/diff_sssd/resources/gen_yamls`: generation YAMLs.
- `src/diff_sssd/generate.py`: canonical synthesis entrypoint.
- `src/diff_sssd/experiment.py`: canonical training entrypoint.

## Prerequisites
- Python environment must contain dependencies in `src/diff_sssd/sssd_standalone/requirements.txt`.
- Run from repository root (`.`).
- `config.yaml` at repository root is used by both scripts.
  - Training save path is `experiment.path.save_root`.
  - Experiment YAML root is `experiment.path.yaml_loc`.
  - Generation YAML root is `experiment.path.gen_yaml_loc`.

## 1) Training procedure (canonical)
1. Create/confirm an experiment yaml in:
   - `src/diff_sssd/resources/exp_yamls/expXXs/expYYYY.yaml`

2. Required keys (observed by loader):
   - `cfg_file`: SSSD config path.
   - `dataset`: `given_split` or `self_split`.
   - `gen_method`: `sssd` (used by model factory).
   - `learning_rate`, `optimizer`, `batch_size`, `n_workers`, `n_iters`, `iters_per_ckpt`.
   - Optional: data/val limits and transfer settings.

3. Run training:
- Single-seed (default):
  ```bash
  python src/diff_sssd/experiment.py --exp 2 --device cuda:0
  ```
- Multi-seed (if seed list is configured in `config.yaml`):
  ```bash
  python src/diff_sssd/experiment.py --exp 2 --device cuda:0 --multirun
  ```
  - Mapping is `exp` -> `resources/exp_yamls/exp{exp//100:02d}s/exp{exp:04d}.yaml`.

4. Optional distributed training:
  - Use `--ddp` with standard torch distributed env vars (`WORLD_SIZE`, `RANK`, `LOCAL_RANK`).
  - Backend can be changed via `--ddp-backend` (default `nccl`).

5. Outputs:
- Training results are saved under:
  - `<experiment.path.save_root>/diff-expXXXX/expYYYY/<timestamp-host>/...`
  - includes model checkpoint `net.pth` and `exp_config.yaml`.
- The checkpoint path to use for synthesis is configured later in `gen_*.yaml` as `ckpt_path`.

## 2) Data synthesis procedure (canonical)
1. Create/confirm generation yaml in:
   - `src/diff_sssd/resources/gen_yamls/genXXs/genYYYY.yaml`

2. Required keys:
- `cfg_path`
- `ckpt_path`
- `output_root`
- `n_gen_train`
- `n_gen_val`
- Optional: `batch_size`, `mode`, `amp_dtype`, `n_per_file`, `compile`, `seed`, label paths.

3. Run generation:
   ```bash
   python src/diff_sssd/generate.py --gen 1 --gpu 0
   ```
- Multi-GPU:
  - Use `--gpu all` or a comma list such as `0,1,2`.
- CPU:
  - `--gpu -1`

4. Output layout (generated by default):
- `<output_root>/<timestamp-host>/train/samples/idx000001.pkl ...`
- `<output_root>/<timestamp-host>/val/samples/idx000001.pkl ...`
- `<...>/gen_config.yaml`
- `<...>/generation_summary.yaml`

5. Label-controlled generation:
- If label paths are set in yaml (`labels_train_path`, `labels_val_path`), generation is conditioned by class labels.
- If absent, generation is unconditional.

6. Optional utilities:
- `src/diff_sssd/trial_generate.py` is an ad-hoc older test script.
- `src/diff_sssd/upsample_pkl.py` performs optional 100Hz→500Hz upsampling for generated PKL outputs.

## Suggested minimal run flow
1. Edit one training yaml (`expXXXX`) and confirm paths.
2. Train with `experiment.py`.
3. Copy the produced `net.pth` path to a generation yaml (`genYYYY`).
4. Run `generate.py`.
5. (Optional) Run `upsample_pkl.py` if downstream data format requires 500 Hz lead-II only.
